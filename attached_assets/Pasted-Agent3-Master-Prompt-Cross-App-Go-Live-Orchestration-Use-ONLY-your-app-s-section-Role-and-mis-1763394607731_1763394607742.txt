Readiness gates (apply to your app)

G0: /health returns JSON with {app, status, version}, 200 OK, no auth required.
G1: /readyz returns 200 OK only when all dependencies are healthy; JSON format; no auth required.
G2: Performance SLO met: P95 ≤ 120 ms on /health and primary read endpoints.
G3: Security headers present: HSTS, CSP, X‑Content‑Type‑Options, X‑Frame‑Options (or equivalent COOP/CORP), Referrer‑Policy, Permissions‑Policy. CORS correctly scoped.
G4: Integrations validated: the contracts your app consumes/serves are verified live with test calls.
G5: Observability: /version exists; trace/request IDs logged; error response schema standardized; metrics endpoint or logging available.
Public apps only: SEO basics (robots.txt, sitemap.xml, canonical tags, meta description, Open Graph, JSON‑LD where relevant).
